# Changelog

All notable changes to Noon Shopping Assistant are documented here.

---

## [1.2.1] — 2026-03-02

### Fixed
- Removed unused `storage` and `activeTab` permissions from `manifest.json` — caused Chrome Web Store rejection on first submission
- These permissions were declared but never called anywhere in the codebase

### Notes
This was the version that got approved and published to the Chrome Web Store. The rejection itself was a useful lesson — Chrome's policy is strict about requesting only the permissions you actually use.

---

## [1.2.0] — 2026-02-24

### Added
- **Context-aware chips** — quick-reply suggestion chips now carry the topic context of their parent message. Clicking "Under AED 500?" after a laptop search sends "Laptops under AED 500" to Claude — not just "Under AED 500?" in isolation
- **Smart show/answer mode** — Claude now decides whether to show product cards or just answer in text. Asking "what should I look for in a laptop?" returns advice, not a product dump. A new `show_products` field in the JSON response schema controls this
- **Unified named comparison flow** — typing "compare iPhone 15 and iPhone 17" now triggers the same compact comparison widget as manually selecting products via checkboxes, instead of a separate heavyweight UI

### Fixed
- **Critical bug: extension not loading on noon.com** — `pA` and `pB` variables were assigned without declaration (`pA = ...` instead of `let pA = ...`) inside a `'use strict'` function, causing a `ReferenceError` on page load. Fixed by properly scoping variables

### Changed
- Welcome screen subheading copy updated to: *"Finally, an AI that actually speaks noon..."*
- Named comparison no longer makes a separate Claude API call for the inline view — specs are fetched only when the user opens the full modal

---

## [1.1.0] — 2026-02-10

### Added
- **Live deal chips** — dynamically fetched deal categories (Mega Deal, Flash Deal, Ramadan offers etc.) shown at the top of the chat on first open
- **Ordinal comparison** — "compare 1st and 3rd" references the last shown product list, no need to type product names
- **Deep spec comparison modal** — full side-by-side spec breakdown with sticky product headers and Claude-generated summary
- **Detail bottom sheet** — tap ✦ on any product card to get AI-powered spec summary and review highlights
- **Compare tray** — select 2–4 products manually via checkboxes, tray appears inline with a "Compare these 2 →" button
- **Expand mode** — panel can be expanded to near-fullscreen with a backdrop blur

### Changed
- Product cards now show discount % badge on image
- Deal tag labels (Ramadan Deal, Mega Deal etc.) shown inline on cards with emoji

---

## [1.0.0] — 2026-01-28

### Initial Release

First working version of the extension. Core features:

- Floating chat panel on noon.com (open/close/settings)
- Conversational product search powered by Claude AI
- Real-time noon.com product data via RSC payload parsing (no external API)
- Product cards with image, price, rating, and direct noon link
- Basic product comparison (named — "iPhone 15 vs Samsung S24")
- Quick-reply chips generated by Claude after each response
- Teaser bubble on first visit with auto-dismiss
- Settings panel — toggle product images, clear conversation
- "Show images" toggle in settings
- Welcome screen with starter chips (Gaming Mouse, Earbuds, Laptops etc.)
- Catalog validation — Claude can only surface products that actually exist in noon's live data, never hallucinated products
